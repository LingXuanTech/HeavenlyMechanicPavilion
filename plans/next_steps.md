# 下一步开发计划（执行摘要）

> 更新日期: 2026-02-11
> 基于当前产品功能迭代现状（Portfolio 强化版）

## 目标

从“可分析”升级到“可执行”：围绕组合分析页面，持续增强配置复用、风险解释与调仓决策能力。

## 优先级清单

## P0（本周必须完成）

1. 增加 **再平衡建议引擎**（当前权重 vs 建议权重 vs 调整幅度）
2. 增加 **调仓动作视图**（增持/减持/保持）与风险暴露变化提示
3. 在 Portfolio API 输出结构化建议字段，前端完成可视化展示
4. 补齐 Portfolio 端到端回归（权重、预设导入策略、建议生成）

## P1（本迭代完成）

1. 预设能力进阶：标签、按标的池分组、搜索与批量管理
2. 增加“组合健康分解卡片”（相关性风险、集中度风险、波动风险）
3. 引入历史快照对比（本次配置 vs 上次配置的评分变化）

## P2（下迭代候选）

1. 跨页面联动：Watchlist 一键推送到 Portfolio 场景模板
2. 基于用户风险偏好生成默认权重方案
3. Portfolio 关键路径 E2E（配置→分析→保存预设→导入复用）

## 里程碑

- **M1（第 2 天）**：再平衡建议后端模型 + API 字段定义完成
- **M2（第 5 天）**：前端调仓建议交互与可视化落地
- **M3（第 7 天）**：Portfolio 回归测试与文档收口完成

## 验收标准

- Portfolio 页面从“分析结果”升级为“可执行建议”闭环
- 预设管理支持冲突处理、导入策略和兼容提示
- 回归测试覆盖关键交互，不出现已知功能回退
- 文档与代码行为一致
